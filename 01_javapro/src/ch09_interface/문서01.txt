레퍼런스 자료형 : 클래스,인터페이스,배열

■인터페이스(interface)
1. 인터페이스는 추상클래스의 일상이다
2. 인터페이스의 모든 메서드는 추상메서드이다(내용 없는 메서드 = 추상메서드)
3. 인터페이스의 메서드 접근제한자 public이다
	생략 하더라도 public이다

4. 인터페이스 변수는 static final이다 (변수를 상수화 하는것)(값 변경 못한다)
	static final을 생략 하더라도 static final이다
	
	ex:
	static final int MAX=100;///클래스 에서는
	int MAX=100;///인터페이스에서는
	값 변경 불가능
	
5. 생성자, static메서드, private에서도 추상화 될 수 수 없다
6. 추상메서드는 상속 받아서 오버라이딩 해야 한다

※
일반 클래스 : 내용 있는 메서드만 존재한다
추상 클래스 : 내용 있는 메서드도있고, 추상 메서드도 함께 존재 한다(짬짜면)
인터페이스 : 모든 메서드가 추상메서드이다.
